DEPLOY — Clone & Configure

Template cloned into deployable engines with unique API keys

❌ PROBLEM
Sharing one workflow across all orgs means shared API keys. When one org exhausts the key's rate limit, ALL orgs go down. No isolation. No customization per client.
✅ SOLUTION
Clone template into separate engines. Each clone gets its OWN API keys. One org hits rate limit? Only their engine is affected. Clones can also be customized without touching others.

ASSIGN — Link to Organization

Engine assigned to org with their KB & Constitution

❌ PROBLEM
A deployed engine is useless until it knows WHOSE data to use. Without linking to org's KB, it has no ICPs, no angles, no playbooks. Without constitution, no guardrails.
✅ SOLUTION
Assign engine to org. Link their Knowledge Base (data) and Constitution (rules). At runtime, engine knows exactly whose config to load. Full multi-tenant isolation.
