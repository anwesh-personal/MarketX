Recommendations

Prioritized phases

Phase 1 — Wire the Data
Foundation
  • MailWizz/IMT Analytics Webhook — POST /api/imt/analytics/events to receive CLICK, REPLY, BOOKED_CALL from MailWizz → insert into analytics_events. Nino — critical handoff from your side.
  • Verify Learning Loop runs; promoted/demoted variants used in workflow resolution
  • Add thumbs up/down or 1–5 rating in chat UI → persist to feedback
Phase 2 — Brain ↔ Workflow
Integration
  • Brain invokes Engine for intents like "write email" or "generate flow"
  • Optional: Engine uses Brain for intent analysis inside workflow nodes
  • Unified User Dashboard: assigned engines + brain + recent runs
Phase 3 — Self-Healing
Learning
  • Extend Learning Loop to update Brain config from outcomes
  • Knowledge Gap Detection: when RAG relevance low, flag gap and optionally queue KB update
  • Strict Grounding Mode: reject or flag when no relevant docs
Phase 4 — User-Level
Granularity
  • Engine clone per user for dedicated config
  • Brain per user (optional overrides)
  • Reseller / Mini Super Admin hierarchy

Handoff for Nino

Key integration points

MailWizz → Axiom Analytics

We need an endpoint to receive engagement events from MailWizz. The Learning Loop reads analytics_events — if MailWizz/IMT never sends events (CLICK, REPLY, BOOKED_CALL, BOUNCE), the loop has no data.

Action: Implement POST /api/imt/analytics/events — Axiom receives, validates, inserts. MailWizz sends via webhook or scheduled sync.

Target User Flow

User signs up (IMT onboarding) → visits MW dashboard → sees assigned flow + brain → emails sent via MailWizz → data (flows, brain, stats) is pure gold.
