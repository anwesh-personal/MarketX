Oraya Brain vs MarketX Brain

Oraya

LOCAL-FIRST
Tauri + Rust + SQLite
Latency: ~0ms
StorageSQLite FTS5 + local vector. Per-agent .db file.
Resonance4-factor: Semantic + Keyword (Jaccard) + Emotional (Valence/Arousal) + Temporal decay.
DreamingSimulated consciousness. States: Active → Dreaming → Asleep. Local pattern consolidation.
MemoryEpisodic, Semantic, Emotional, Procedural. Personal context only.
SafetyEmotional regulation. Adjusts based on internal state vectors.
IsolationFile-based. Each agent owns its .db file on disk.
PROS
  • + 100% data sovereignty
  • + Zero network latency
  • + Free to run (local HW)
  • + Rich emotional nuance
CONS
  • - Single user only
  • - Heavy on local CPU/RAM
  • - No shared org memory
  • - File loss = brain gone

MarketX (Axiom)

CLOUD-NATIVE
Next.js + Supabase + pgvector
Scale: Multi-Tenant
StoragePostgres (Supabase) + pgvector. ~80 tables. RLS per org.

MemoryKB, Brand Voice, Constitution, Past Campaigns, Performance Metrics. Business context.

  • + Background workers offload
  • + Enterprise-grade safety
CONS

  • - Complex distributed system

Security Audit — Honest Status

From launch_audit.md. 18 CRITICAL, 24 WARNING. These need fixing before go-live.

CRITICAL Missing Auth on Brain Endpoints
●brain/embeddings — GET+DELETE with ZERO auth. Anyone can list/delete embeddings.
●brain/templates — GET with no auth. Lists all templates.
●brain/agents — GET+PATCH with no auth. Anyone can list/update any agent.
●brain/config — GET with no auth. Leaks brain config.
Fix: Add getRequestContext() auth check + org_id filter to each of these routes.
CRITICAL Superadmin Routes — No JWT Verification
●superadmin/users/impersonate — NO AUTH. Anyone can impersonate any user!
●superadmin/vps/deploy — NO AUTH. Can restart all workers.
●superadmin/ai-chat — No JWT. Full LLM access with service role key.
●superadmin/ai-providers/*/test — No auth. Accepts raw API keys in body.
●superadmin/redis/status — No auth. Exposes Redis internals.
Fix: Create requireSuperadmin() middleware. Apply to all /api/superadmin/* routes.
CRITICAL Tenant Isolation Bypass
●brain/embeddings — SELECT * with no org_id filter. Cross-tenant leak.
●brain/agents — SELECT/UPDATE with no ownership check.
●brain/templates — No partner_id filter.
Fix: Every query MUST include .eq('org_id', orgId) or .eq('partner_id', orgId).
CRITICAL Hardcoded JWT Secret Fallback
●'axiom-jwt-secret-change-in-production' used as fallback in 2 files.
Fix: Remove fallback. App should throw on startup if JWT_SECRET not set.
WARNING Missing DB Migrations
●brain_beliefs — Referenced in get_brain_health tool + mastery-agent-worker. No CREATE TABLE migration found.
●brain_decisions — Written to by mastery-agent-worker. No migration found.
●brain_knowledge_items — Read by mastery-agent-worker. No migration found.
Impact: get_brain_health tool and mastery-agent-worker will throw at runtime. Need migration.

Estimated fix time: Security fixes ~4-6 hours. Missing migrations ~1 hour. Infra setup ~1 day. After that, it's go-live ready.
