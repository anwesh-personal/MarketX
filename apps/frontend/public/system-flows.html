Request Body

Create Engine Bundle

Request Body
name: string *
brain_template_id: UUID (from step 4)
workflow_template_id: UUID
tier: echii|pulz|quanta
default_api_key_mode: platform|byok|hybrid
— Then PATCH to add: —
agents_config: [{role, name, llm, prompts, tools, rag}]
default_llm: {provider, model, temperature, max_tokens}
UI: 3-Tab Form
Tab 1: Basics
Name, Brain Template, Workflow, Tier, API Key Strategy
Tab 2: Default LLM
Provider, Model, Temperature, Max Tokens
Tab 3: Agents
Add agents with role, LLM, prompts, tools, RAG, memory config
Bundle = Brain + Workflow + Agents + Config. This is the "product" you sell to clients.
POST /api/superadmin/engine-bundles/{id}/deployHAS AUTH

Deploy Bundle to Organization

Request Body
org_id: UUID *
assigned_user_id: UUID (null = org-wide)
api_key_mode: platform|byok|hybrid
byok_keys: {openai?, anthropic?}
deployment_notes: string
Deploy Sequence (6 steps)
1 Validate bundle + org + no duplicate
2 Build snapshot (freeze bundle state)
3 Clone → Insert brain_agents per role
4 Generate API key axm_live_xxx
5 Insert engine_instances
6 Insert engine_bundle_deployments (audit)
Step 3 is THE critical step — it creates brain_agents records. Without these, Brain Chat returns "No active brain agent deployed." This is the bridge between Superadmin and Member.

Brain Chat — The Core Product

Load Sequence
GET /api/brain/runtime
getActiveBrainRuntime(orgId)
Reads brain_agents where org_id, user_id IS NULL, status active
If no brain_agents → "No active brain agent deployed"
Chat Flow (POST /api/brain/chat)
Auth → org_id
ToolLoader.getAgentTools() — from brain_tools + min_tier
PromptAssembler — 7 layers + RAG + memories + constitution
BrainOrchestrator.handleTurn() — up to 20 turns
AIProviderService.generateChat() → provider chain
Tool calls → HallucinationInterceptor → MarketXToolExecutor
Save to messages, log metrics, SSE stream
Requires: (1) brain_agents record (from bundle deploy), (2) AI provider configured (from step 3), (3) brain_tools rows in DB. Without ALL three, Brain Chat fails.

Gaps & Broken Links

Every issue found by tracing actual code.

CRITICALOnboarding API inserts wrong columns into knowledge_bases
API inserts content, status, is_primary — but knowledge_bases table has data (JSONB), stage, no is_primary. Insert will fail.
File: api/onboarding/complete/route.ts line 43-48
CRITICALWriter form and API are out of sync
Form sends kb_id, tone, length. API ignores kb_id, uses Brain KB from brainKBService. API expects angle_class, email_count, flow_goal — form doesn't send these.
File: writer/new/page.tsx vs api/writer/execute/route.ts
CRITICALKB APIs have zero auth
All /api/kb/* routes use SERVICE_ROLE_KEY only. No user authentication, no org scoping. Anyone can CRUD any org's knowledge bases.
File: api/kb/route.ts, api/kb/[id]/route.ts
CRITICALSuperadmin org/user routes have no auth
POST /api/superadmin/organizations and POST /api/superadmin/users have zero authentication. Anyone can create orgs and users.
File: api/superadmin/organizations/route.ts, api/superadmin/users/route.ts

agent_templates table missing (6 seed agents), 2 brain tools not synced, infra config extensions missing, signal_event idempotency missing + 4 brain tools.
