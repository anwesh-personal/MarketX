Email Infrastructure
Week 2–3
Replaces Nino
Own your own email sending. Mailgun as API layer. Webhook receiver to capture all signals back.

Phase 4
Market Writer Core
Week 4–7
The Core Product
Belief-bound email generation. Brief immutable. Champion vs Challenger. 7 canonical angles. Flow extension rules.
Build Brief generation service (one Brief = one ICP)
Champion + Challenger belief creation (80/20 allocation start)
Generate Flow A (Champion) + Flow B (Challenger): emails 1–4
Subject A/B per email, single CTA, plain text
Flow extension: 5–8 (clarification), 9–12 (confirmation), post-12 (reflection)
Build email destination pages (Angle × ICP × Buyer Stage)
Implement 7 canonical angle classes

Phase 5
Belief Promotion Engine
Week 6–8
The Intelligence Layer
Automate belief testing and promotion. HYP→GW state machine. 5-gate hybrid gating. No manual intervention needed.
Daily signal aggregation worker (reads signal_event per belief)
Confidence score calculation formula
Hybrid gate evaluator: 5 conditions must ALL pass
Promotion state machine: HYP→TEST→SW→IW→RW→GW
Allocation adjustment: stepwise, explorer % always maintained
Log all transitions in belief_promotion_log
Suppression (not deletion) when belief fails
