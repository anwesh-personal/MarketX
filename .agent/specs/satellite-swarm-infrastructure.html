Section 05

DNS Configuration (Per Domain)

Every domain MUST have all 5 records configured correctly before any email is sent. This is non-negotiable for deliverability.

SPF
@ (root domain)
v=spf1 ip4:<SERVER_IP> ~all
DKIM
mw._domainkey
v=DKIM1; k=rsa; p=<PUBLIC_KEY>
DMARC
_dmarc
v=DMARC1; p=quarantine; rua=mailto:dmarc@domain.com; fo=1
PTR (Reverse DNS)
<IP Address>
mail.yourdomain.com
MX
@ (root domain)
10 mail.yourdomain.com
⚠ CRITICAL: Reverse PTR records must be set at the hosting provider level (not domain registrar). Each dedicated IP needs its own PTR record matching the sending domain. This is the #1 missed step that kills deliverability.

Provider Recommendations:
• OVH — Best for bulk mail. Port 25 open by default. Cheap extra IPs (~$2/IP/mo). Good rDNS control.
• Contabo — Very cheap. Port 25 open. Manual rDNS via support ticket.
• Hetzner — Excellent hardware. Port 25 blocked by default (must request unblock, strict policy).
• Linode — Good but port 25 restricted for new accounts (support request needed).
• DigitalOcean / AWS — Port 25 blocked. NOT recommended for self-hosted mail.

Postfix config: sender_dependent_default_transport_maps routes each domain's mail through its assigned IP. This is a standard Postfix feature — any experienced mail admin will know how to set this up.
