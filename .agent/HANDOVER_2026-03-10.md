- **Next step for production:** Either add a migration that creates `belief` (or the full RS:OS core) before/with 028, or make `belief_id` a plain UUID column without FK until RS:OS is introduced.

### 7b. "I see nothing in superadmin / user UI" (troubleshooting)

**Most likely cause:** The app reads from **Supabase** (via `NEXT_PUBLIC_SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY`). If you ran migrations on a **different** database (e.g. local Postgres with psql), that DB has the tables and data — but the app never talks to it. So the UI shows zeros, empty lists, or "Failed to load stats".

**What to do:** (1) Run migrations **in the Supabase project** (Dashboard → SQL Editor or `supabase db push`) so the app's DB has the tables. (2) After login, if the stats API fails the dashboard shows a red banner with the error — use it to confirm (e.g. missing env, relation does not exist). (3) Superadmin login uses `platform_admins` in the same DB; default `anweshrath@gmail.com`. (4) User UI uses the same Supabase; create an org via superadmin so there is data.
