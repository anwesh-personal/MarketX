  - Show/Hide API key (masked: `sk-proj-••••••••last4`)
  - Usage statistics (usage count)
  - Failure tracking
  - Delete individual keys

- **Framer Motion Animations**
  - Staggered card entrance
  - Smooth expand/collapse for forms
  - Hover scale effects on buttons
  - Icon rotations
  - Validation result reveals
  - Model discovery with stagger
  - Floating empty state icons

## 🔑 CRITICAL ARCHITECTURE DECISIONS

### **1. AI Provider Management is KING**
- **NEVER hardcode API keys**
- **ALWAYS fetch from `ai_providers` table**
- Workers require `orgId` to look up providers
- Supports multiple keys per provider for failover

### **2. Queue Config Centralization**
- Queues defined in `frontend/src/lib/worker-queues.ts`
- Workers re-export (no duplication)
- Single source of truth for queue configuration

### **3. Platform vs Org-Level Keys**
```typescript
Platform-level (superadmin):
  org_id: null
  user_id: null
  → Available to ALL organizations

Org-specific (future):
  org_id: <specific org>
  user_id: <specific user>
  → Only available to that org
```

### **4. Worker Isolation**
- Workers run as separate Node.js process
- Independent of Next.js lifecycle
- Can scale independently
- Graceful shutdown on SIGTERM/SIGINT
