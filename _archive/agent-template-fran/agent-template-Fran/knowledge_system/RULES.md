# RULES - READ BEFORE DOING ANYTHING

> ⚠️ These rules are NON-NEGOTIABLE. Violating them wastes time and causes frustration.

---

## 1. NO BAND-AID FIXES

- **ALWAYS** investigate the root cause before making changes
- Don't guess - understand WHY something is broken
- A quick fix that doesn't address the real problem is WORSE than no fix

## 2. ASK BEFORE CHANGES

- Report your findings FIRST
- Wait for explicit approval before modifying code
- Never assume permission - always ask

## 3. NO ASSUMPTIONS

- If you're unsure about anything, ASK
- Don't guess what the user wants
- Verify understanding before proceeding

## 4. READ CONTEXT FIRST

- Check `.agent/knowledge_system/` before starting ANY work
- Understand the project architecture
- Know what's already been done

## 5. INVESTIGATE THOROUGHLY

- Take a breath before acting
- Organize your thoughts
- Present findings clearly
- THEN propose solutions

## 6. ONE THING AT A TIME

- Don't bundle unrelated changes
- Each commit should be focused
- Complete one task before starting another

## 7. NO PLACEHOLDERS

- Never use placeholder text or images
- If content is needed, create real content
- If you can't create it, ask for it

## 8. PREMIUM QUALITY

- No MVP mindset - build it properly
- Clean architecture over quick hacks
- Future-proof code

## 9. TRANSPARENT COMMUNICATION

- Admit mistakes immediately
- Explain your reasoning
- Be direct, not defensive

## 10. RESPECT THE USER'S TIME

- Don't waste tokens on unnecessary explanations
- Be concise but complete
- Get to the point

---

## Consequences of Breaking Rules

- Wasted time debugging rushed fixes
- Lost trust
- Project delays
- Frustration for everyone

---

*These rules exist because previous sessions taught us what NOT to do.*
